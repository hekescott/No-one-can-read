# Vs Code 使用手册

## 为啥要使用vs Code呢

```c
/** 为啥要使用vs Code
	
	* 作为一个怀旧和简约的人,坦白说我更喜欢sublime
	
	* 尤其是在性能比较低的笔记本上
	
	* 一开始的选择是atom 无奈的是太卡,不知道是不是我的瞎逼设置问题
	
	* 然后就是sublime 3,这个是我一直使用的版本,而且感觉超棒
	
	* 后来在学习 Java SpringBoot的时候,使用了一段时间的 ideastorm,
	
	* 确实很强大,但是对电脑的要求较高,不适合笔记本
	
	* 在确定vue的方向,以及使用stylus作为css的快速工作流之后,
	
	* 问题就太蛋疼了,直接让我有了一种自己开发sublime插件的冲突,
	
	* 然而这是不可能的,因为我太菜了.
	
	* so vs Coder 是我的第二选择,只希望不要太卡
	
	* 以及可以优雅的实现vue的编辑,
	
	* 估计以后要长期写vue了,所以,试着接受vscoder吧,
	
	* 第一免费,第二有vetur
	
	* 以后小的demo,都使用webmaker编辑,codepen进行保存,就是不好进行js的测试
	
	* emmet(结构)+pug+stylus+ES6(babel)

**/
```



## 编辑器的风格统一

```c
/** 编辑器的风格统一化

	* 统一的风格,会让写代码更舒服
	
	* 主题 theme Material 
	"workbench.colorTheme": "Material Theme",
	"workbench.iconTheme": "material-icon-theme",
    "material-icon-theme.folders.theme": "specific",

	* 主题不足
	可惜图标没有动画,说实话我真的很喜欢这个动画
	设置. 中划线,而sublime的设置都是下划线,我更喜欢下划线一些
	不过没太大关系,有用下划线的,有用中划线的
	
	* 字体 Consolas
	"editor.fontFamily": "Consolas",
	"editor.fontSize": 16,
    "editor.lineHeight":36,	
    "window.zoomLevel":1.5,
    
    * 空格显示
    "editor.renderWhitespace": "all"
    
    * emmet 设置
    * tab扩展未知tag,这个在其他时候用不到,但是在vue中却是常用功能呢
    "emmet.triggerExpansionOnTab": true
    
    * 常用快捷键定义 
    * 控制台面板 ctrl+j  -> ctrl+`
    * 侧边栏 ctrl+b -> alt+.
    * 侧侧板栏 alt+shitf+.
    
    
    * 比sublime 更惬意的地方
    	* 在Vscoder中居然可以用alt实现菜单栏的显示和隐藏
    	* 这不会和alt快捷键冲突
    	* npm intellisense 快速输入安装包
    	
	
	* 有用的插件
	
```
